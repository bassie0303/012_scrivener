## A. 学習体験

### A-1. AI解説の自動生成（2層構成）★大きめ
**状態: アプリ表示＋生成スクリプトは実装済み（2026-06-22）。実データへの一括生成（API実行）は未実施。**
- アプリ: 解答後に「総合解説 → 各肢の解説」を表示。○×一問一答ではその肢の肢別解説を表示。サンプル問題には解説入り。
- 生成: `data/pipeline/generate_explanations.py`（Opus 4.8・構造化出力・再開可能）。要 `ANTHROPIC_API_KEY`。
  - 実行: `python generate_explanations.py ../processed/questions.json`（試運転 `--limit 3`）→ `web/public/data/questions.json` にコピー、各端末で取り込み直し。
  - 概算 約$6（≈1,000円）/1回・331問。Batches APIで半額化は将来候補。

各問に解説を付け、解答後に表示する。解説はAI生成の独自コメント＝著作権上問題なし。
**保存先は端末ローカルの `questions.json`（同期・公開しない）。**

- **5択(tantou5) は2層**:
  - `explanation`（総合）… テーマ・論点、5肢を見比べた正誤の分かれ目、正解肢が際立つ理由、引っかけ注意（100〜150字）
  - `choice_explanations`（肢別）… 各肢が○/×の根拠を条文・判例でピンポイント（各50〜80字）。○×一問一答でもそのまま使える
- **多肢(tashi)** … 総合1本（任意でア〜エの根拠）
- **記述(kijutsu)** … 解答の法的筋道1本
- **生成方法**: 1問1コールで総合＋肢別を**構造化出力**(`output_config.format`)で一括取得（設問・全選択肢・正解を一度に見せ、総合と肢別の整合を取る）。
- **モデル/コスト**: 推奨 **Opus 4.8**（法的正確性重視）。331問・肢別込みで **約$6（≈1,000円）/1回限り**。Batches APIで半額、prompt cachingで共通指示はほぼ無料。Sonnet 4.6 でも実用可。
- **実装**: `data/pipeline/generate_explanations.py` を追加 → `questions.json` に `explanation`/`choice_explanations` を追記。アプリは解答後に「総合 → 各肢」の順で表示。

### A-1b. 解説表示の改善＋候補マーク（実装 2026-06-22）
- 各肢/各記述の ○× を**解説テキストの判定語**（正しい/妥当→○緑、誤り/妥当でない→×朱）で表示（自己整合）。
- **組合せ問題**は「各肢」ではなく**記述ア〜エ別の解説**(`statement_explanations`)＋「正解の組合せ」を表示。
  `generate_explanations.py` が組合せ問題を記述別で生成（正解の組合せと整合）。既存の組合せ77問は
  `statement_explanations` 未生成なので再実行で作り直し（通常5択は対象外＝据え置き、約$1.5）。
- **候補マーク**: 各選択肢に ○候補/×候補/解除。×候補は打ち消し表示。最終解答とは別の作業メモ（問題ごとにリセット）。

### A-2. 直前に見ていた問題を再開する ★小（実装済み 2026-06-22）
一度アプリを閉じて再度開いたとき、**前回最後に見ていた問題を開く**。
実装: `lastQuestionId` を localStorage 保存、起動時に現デッキ内を id 検索して位置復元（1回のみ）。

- 最後に表示した `question_id`（＋モード/フィルタ状態）を保存（UI設定なので localStorage で可。文字サイズ・フィルタ設定と同様）。
- 起動時、現在のデッキ（フィルタ適用後）にその問題が含まれれば、その位置 `idx` に復元。含まれなければ先頭。
- 注意: デッキは年度/分野/学習対象フィルタで変わるので、`question_id` で探して位置を合わせる（`idx` 直接保存は不可）。

### A-3. 問題ごとの正誤を星で表示 ★小（実装済み 2026-06-22）
問題カード上部に、その問題の**過去成績を星の数で**示す。
実装: 過去正解＝白星☆(朱)・過去誤答＝黒星★(藍墨)。上限10＋「+N」。○×は肢単位の履歴で表示。

- 例: **過去正解＝白星 ☆、過去誤答＝黒星 ★**（または色分け）。
- データ源は履歴 `history[question_id]`: 白星 = `correct_count` 個、黒星 = `attempts - correct_count` 個。
- 既存の「この問題：N回中M正解」表示の隣に視覚版として置く。
- 多すぎる場合の対策: 上限（例: 直近5回ぶん）か「☆×3 ★×1」のように数字併記でコンパクト化。
- ○×一問一答は肢単位(`r7-2-3`)で履歴が別なので、星も肢単位 or 親問題で集約するか方針を決める。

## C. データ品質

### C-1. 分野分類の精度向上（`classify_field.py`）
番号レンジ＋設問文キーワードで概ね正確だが、稀な誤発火がある
（例: 「裁判制度／簡易裁判所」で "代理" 等の語に反応）。行政法のサブ分類
（行政手続/不服審査/事件訴訟/国家賠償/地方自治）も将来候補。

### C-3. 選択肢が抽出できていない5択が3問ある ★小（実装済み 2026-06-22）
`r2-7 / r4-49 / r5-57`（組合せ問題）はパーサーが（注）以降を選択肢でなく脚注として拾い choices が空だった。
対応: `data/pipeline/patch_questions.py` が各 `*_full.json` の raw 末尾「1 ア・イ」形式から5択を復元して付与。
→ 通常の正解付き5択として出題可能に（answer は r2-7=4 / r4-49=4 / r5-57=1）。

### C-2. R6-34（全員正解の没問）の扱い（実装済み 2026-06-22）
公式「全員正解」で単一正解なし。`patch_questions.py` が `all_correct=true` で参考収録（分野=民法を付与・注記入り）。
アプリ: any肢で正解扱い、「全員正解（公式に没問）」表示、○×変換からは除外。

＊どちらも `patch_questions.py`（local の *_full.json から取得）で `questions.json` をパッチ。生成済み解説は保持。
  反映: `cp ../processed/questions.json ../../web/public/data/questions.json` → 各端末で取り込み直し。
  補修した4問の解説を選択肢込みで作り直したい場合は `generate_explanations.py --force`（任意・要API）。
